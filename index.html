— Under the Hood

How it all connects

A lean IoT pipeline — from a physical door sensor to your browser in seconds.

01
Door sensor triggers ESP32
A reed switch or PIR on the toilet door fires an interrupt on the ESP32. It captures the event type (OPEN / CLOSE) and increments the running counters.
02
ESP32 writes to Firestore
Over Wi-Fi, the ESP32 posts a new document to Firebase Firestore with 5 fields: event, openCount, closeCount, device, and a server timestamp.
03
Dashboard fetches & renders
AirLoo calls the Firestore REST API every 10 seconds, parses up to 300 documents, and renders charts, KPIs, heatmaps, and alerts — all client-side.
04
Smart alerts fire automatically
Rules run on the latest data: unclosed doors, high session counts, long idle periods — giving facility managers instant visibility.
